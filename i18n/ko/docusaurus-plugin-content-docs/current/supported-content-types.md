# 컨텐츠 유형과 사용 예시

이 섹션에서는 FLOWER가 지원하는 콘텐츠 유형과 광고 서비스 활용 예시를 설명합니다.

## 실시간 채널

### 본 스트림 대체 광고

이 광고 유형은 광고 마커(예: SCTE-35)를 활용하여 본 콘텐츠 스트림 내 기존 광고를 대체합니다. SCTE-35 기반 대체 광고는 두 전달 방식 모두에서 지원되며, 처리 방식이 다릅니다:

*   **Unicast (HLS/DASH)**: SDK가 재생목록 매니페스트(m3u8 또는 mpd)를 가공하여 광고 마커를 감지하고 스트림 내 광고 구간을 대체합니다.
*   **DTH / Multicast**: SDK가 TS 스트림에서 SCTE-35 큐를 감지하고, Ad Agent가 로컬에 캐싱된 광고 소재를 광고 레이어에서 재생합니다.

#### 사용 예시
*   기존 광고 구간을 자체 광고로 대체하여 수익을 창출합니다.
*   FAST 채널의 콘텐츠 사이에 광고를 삽입하여 수익을 창출합니다.
*   라이브 스포츠 중계 시 광고를 대체하여 수익을 창출합니다.

### 채널 진입 광고

시청자가 본 콘텐츠 스트림에 진입하기 전에 재생되는 광고입니다. 시청자가 실시간 콘텐츠를 시청하려고 하면 먼저 진입 광고가 표시된 후 본 스트림으로 전환됩니다. Unicast (HLS/DASH) 전달 방식에서 사용 가능합니다.

#### 사용 예시
*   무료 채널에 채널 진입 광고를 설정하여 수익을 창출합니다.
*   구독형 OTT 서비스에서 광고 요금제를 통해 광고 수익을 창출합니다.

### 아날로그 큐 및 푸시 기반 광고

DTH / Multicast 전달 방식에 특화된 광고 유형으로, 백엔드 시스템이 클라이언트 디바이스의 광고 재생을 트리거합니다.

*   **아날로그 큐**: H/E(Head-End) 시스템에서 감지된 광고 큐가 Anypoint 플랫폼을 통해 디바이스로 전달되어 광고 재생을 시작합니다.
*   **푸시 기반 (가상 큐)**: 백엔드가 방송 프로그램 일정표를 기반으로 가상 큐를 생성하여 푸시 서버를 통해 디바이스에 전송합니다.

#### 사용 예시
*   H/E 시스템이 감지한 방송 광고 구간에 광고를 삽입합니다.
*   스트림 내 마커 없이 프로그램 일정표 기반으로 광고 구간을 예약합니다.

## VOD

FLOWER는 VOD 콘텐츠에 대해 다양한 광고 형식을 지원합니다:
*   Pre-roll 광고: 본 콘텐츠 시작 전에 광고를 삽입합니다.
*   Post-roll 광고: 본 콘텐츠 종료 후에 광고를 삽입합니다.
*   Mid-roll 광고: 콘텐츠 내 일정 간격 또는 지정된 광고 삽입 지점에 광고를 삽입합니다.

### 사용 예시
*   장편 VOD 콘텐츠에 pre-roll, mid-roll, post-roll 광고를 삽입하여 수익을 창출합니다.
*   광고 삽입 지점을 활용하여 자연스러운 구간에 광고를 배치합니다.

## 웹페이지 / 앱

FLOWER를 사용하면 웹페이지와 앱에 비디오 광고를 원활하게 통합할 수 있습니다.

### 사용 예시
*   포털 사이트 메인 페이지 우하단에 300x400 크기의 비디오 광고를 삽입합니다.
*   블로그 게시글 중간에 비디오 광고를 삽입합니다.
*   게임 레벨 사이에 전면 광고를 삽입합니다.
*   앱 진입 전 광고 시청을 요구하여 수익을 창출합니다.
